github setting_manual

집과 회사 양쪽에서 동일한 VS Code 환경을 구축하고 GitHub로 소스 코드를 관리하는 것은 개발 생산성을 높이는 아주 좋은 방법입니다.

이를 위해 **기본적인 연동 방법**부터, 회사와 집을 오가며 **동일한 환경을 유지하는 핵심 팁**까지 단계별로 안내해 드리겠습니다.

---

### 1단계: 필수 프로그램 설치 (집/회사 공통)

가장 먼저 두 PC 모두에 아래 프로그램들이 설치되어 있어야 합니다.

1. **Git 설치:** [Git 공식 홈페이지](https://git-scm.com/)에서 운영체제에 맞는 Git을 다운로드하여 설치합니다. (설치 시 기본 옵션으로 계속 'Next'를 눌러 진행하시면 됩니다.)
2. **VS Code 설치:** [VS Code 공식 홈페이지](https://code.visualstudio.com/)에서 설치합니다.

---

### 2단계: VS Code와 GitHub 계정 연동하기

터미널에서 명령어를 치는 방법도 있지만, VS Code에 내장된 기능을 사용하는 것이 가장 쉽고 직관적입니다.

1. VS Code를 실행합니다.
2. 왼쪽 사이드바 맨 아래에 있는 사람 모양 아이콘(Accounts)을 클릭합니다.
3. **'Turn on Settings Sync...' (설정 동기화 켜기)** 또는 'Sign in to Sync Settings'를 클릭합니다.
4. 로그인 방식으로 'Sign in with GitHub'를 선택합니다.
5. 웹 브라우저가 열리며 GitHub 권한 부여 창이 뜹니다. 승인(Authorize)을 완료하면 VS Code와 GitHub가 연동됩니다.

---

### 3단계: 소스 불러오고(Clone) 수정 후 저장하기(Push)

이제 GitHub에 있는 코드를 내 PC로 가져오고, 수정한 뒤 다시 올리는 과정입니다.

#### A. 코드 불러오기 (Clone)

1. VS Code에서 `F1` 키(또는 `Ctrl + Shift + P`)를 눌러 명령어 팔레트를 엽니다.
2. `Git: Clone`을 검색하여 선택합니다.
3. 'Clone from GitHub'를 클릭하거나, 위쪽 입력창에 복사해 둔 **GitHub 레포지토리 URL**을 붙여넣습니다.
4. 소스 코드를 저장할 내 PC의 폴더를 선택합니다.
5. 우측 하단에 'Open' 알림이 뜨면 클릭하여 프로젝트를 엽니다.

#### B. 코드 수정 및 저장 (Commit & Push)

1. 코드를 자유롭게 수정하고 저장(`Ctrl + S`)합니다.
2. 왼쪽 사이드바에서 세 번째에 있는 **소스 제어(Source Control)** 아이콘(가지 모양)을 클릭합니다.
3. 수정된 파일 목록이 보입니다. 파일 옆의 `+` 아이콘(Stage Changes)을 눌러 변경 사항을 추가합니다. (이 과정을 'Staging'이라고 합니다.)
4. 위쪽 텍스트 입력창에 어떤 부분을 수정했는지 간단한 메모(Commit message)를 적습니다. (예: `기능 추가: 로그인 화면 구성`)
5. **`Commit`** 버튼을 클릭합니다. (내 PC에 변경 이력을 저장하는 과정입니다.)
6. 마지막으로 **`Sync Changes`** (또는 `Push`) 버튼을 누르면 GitHub 서버로 코드가 업로드됩니다.

### 4단계: 회사 PC 환경 설정 시 주의사항

회사 PC에서도 1~3단계 과정을 똑같이 진행하시면 됩니다. 단, 회사 환경 특성상 몇 가지 주의할 점이 있습니다.

* **회사 방화벽 및 보안 프로그램:** 회사 보안 정책상 GitHub(외부망) 접속이나 SSH 포트가 막혀 있을 수 있습니다. 저장소 주소를 복사할 때 SSH 방식 대신 **HTTPS 방식의 주소**를 사용하시는 것을 권장합니다. 오류가 발생한다면 사내 사내 IT/보안 부서에 문의해야 할 수 있습니다.
* **Git 계정 정보 분리 (선택 사항):**
개인 프로젝트는 개인 이메일로, 회사 프로젝트는 회사 이메일로 기록(Commit)을 남겨야 할 때가 있습니다. 회사 프로젝트 폴더를 열고 VS Code 터미널(`Ctrl + ``)에서 아래 명령어를 입력해 해당 폴더에서만 작동하는 정보를 입력할 수 있습니다.
```bash
git config user.name "내이름"
git config user.email "회사이메일@company.com"

```

### 💡 핵심 팁: 집과 회사의 VS Code "완벽 동기화" 하기

"동일하게 사용하려고 한다"는 목적을 달성하기 위한 가장 중요한 설정입니다. 집 PC에서 설치한 확장 프로그램(Extensions), 단축키, 테마 등을 회사 PC에서도 똑같이 쓰려면 **설정 동기화(Settings Sync)** 기능을 사용하세요.

1. 집 PC의 VS Code 왼쪽 아래 **톱니바퀴(Settings) 아이콘**을 클릭합니다.
2. `Turn on Settings Sync`를 클릭합니다.
3. 동기화할 항목(Settings, Keyboard Shortcuts, User Snippets, Extensions, UI State)을 모두 체크하고 `Sign in & Turn on`을 누릅니다.
4. GitHub 계정으로 로그인합니다.
5. **회사 PC**에서도 똑같이 톱니바퀴 > `Turn on Settings Sync`를 누르고 같은 GitHub 계정으로 로그인합니다.
6. 잠시 기다리면 집 PC의 테마, 단축키, 확장 프로그램들이 회사 PC에 자동으로 설치되고 동일한 환경이 구성됩니다.
